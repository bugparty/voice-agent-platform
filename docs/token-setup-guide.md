# Twilio Access Token Setup Guide

## Problem

If you see this error:
```
AccessTokenInvalid (20101): Twilio was unable to validate your Access Token
```

it means the Web token configuration is incorrect.

---

## Solution: Configure Environment Variables

The token is generated by the **media-service** backend, and it requires 3 Twilio credentials.

### Step 1: Create an API Key (in Twilio Console)

1. Sign in to [Twilio Console](https://console.twilio.com/)
2. Go to **Account → API Keys & Tokens**
3. Click **Create API Key**
4. Fill out the form:
   - **Friendly Name**: `voip-agent-api-key` (or any name)
   - **Key Type**: select **Standard**
5. Click **Create**
6. ⚠️ **Copy and save immediately**:
   - **Key SID**: `SKxxxxxxxxxxxxxxxxxxxxxxxxxxxx`
   - **Secret**: `xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx` (**shown only once!**)

### Step 2: Create a TwiML App (in Twilio Console)

1. In Twilio Console, go to **Voice → TwiML Apps**
2. Click **Create new TwiML App**
3. Fill out the form:
   - **Friendly Name**: `voip-agent-web-join`
   - **Voice Configuration - Request URL**: `https://YOUR_PUBLIC_BASE/twiml/webJoin`
     - ⚠️ **Important**: This URL must be **publicly accessible** (not localhost)
     - If you use ngrok: `https://xxxx.ngrok.io/twiml/webJoin`
     - This URL must match the `PUBLIC_BASE_URL` environment variable
   - **HTTP Method**: `POST`
4. Click **Save**
5. Copy the **App SID**: `APxxxxxxxxxxxxxxxxxxxxxxxxxxxx`

> 💡 **Debug tip**: When the Web client joins a Conference, Twilio sends a POST request to this URL.
> If you do not see `/twiml/webJoin` requests in media-service logs, your TwiML App configuration is likely incorrect.

### Step 3: Configure Environment Variables

Create or edit the `.env` file in the `apps/media-service` directory:

```bash
# ============================================
# Twilio basic configuration (required)
# ============================================
TWILIO_ACCOUNT_SID=ACxxxxxxxxxxxxxxxxxxxxxxxxxxxx
TWILIO_AUTH_TOKEN=your_auth_token_here
TWILIO_FROM_NUMBER=+1234567890

# ============================================
# Phase 2: Web Join Token configuration (required!)
# ============================================
TWILIO_API_KEY=SKxxxxxxxxxxxxxxxxxxxxxxxxxxxx
TWILIO_API_SECRET=your_api_secret_here_only_shown_once
TWILIO_TWIML_APP_SID=APxxxxxxxxxxxxxxxxxxxxxxxxxxxx

# ============================================
# Other configuration
# ============================================
FIXED_TO_NUMBER=+16198597172
PUBLIC_BASE_URL=https://your-ngrok-url.ngrok.io
```

### Step 4: Restart media-service

```bash
# Press Ctrl+C to stop the service
# Then start it again
pnpm --filter media-service start
```

## Verify Configuration

### Method 1: Check logs on startup

After starting media-service, you should **not** see this warning:

```
# ✅ Correct configuration - no warning

# ❌ Missing configuration - you will see:
[media-service] Twilio API credentials not configured for token generation
```

### Method 2: Test the token endpoint

Test with curl or Postman:

```bash
curl -X POST http://localhost:4001/token \
  -H "Content-Type: application/json" \
  -d '{"identity": "test_user", "sessionId": "test_session"}'
```

**Expected response:**
```json
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "identity": "test_user",
  "sessionId": "test_session"
}
```

**Error response (missing configuration):**
```json
{
  "error": "Token generation not configured. Please set TWILIO_API_KEY, TWILIO_API_SECRET, and TWILIO_TWIML_APP_SID"
}
```

### Method 3: Test from the Web client

1. Click the **Call** button (PSTN call)
2. Click the **Join Conference** button
3. Check the browser Console:

**Success:**
```
[Web UI] Step 3: Requesting token from backend...
[Web UI] Token response received { hasToken: true, hasError: false }
[Twilio Device] ✓ Device registered successfully
```

**Failure:**
```
[Web UI] Token response received { hasToken: false, hasError: true }
[Web UI] Token error: Token generation not configured...
```

## Complete .env Template

```bash
# ============================================
# Twilio account info (from Console)
# ============================================
# Account SID: https://console.twilio.com/
TWILIO_ACCOUNT_SID=ACxxxxxxxxxxxxxxxxxxxxxxxxxxxx

# Auth Token: https://console.twilio.com/
TWILIO_AUTH_TOKEN=your_auth_token_here

# Your Twilio phone number
TWILIO_FROM_NUMBER=+1234567890

# ============================================
# Phase 2: Web Join configuration
# ============================================
# Create via: Account → API Keys & Tokens → Create API Key
TWILIO_API_KEY=SKxxxxxxxxxxxxxxxxxxxxxxxxxxxx
TWILIO_API_SECRET=your_secret_only_shown_once

# Create via: Voice → TwiML Apps → Create new TwiML App
TWILIO_TWIML_APP_SID=APxxxxxxxxxxxxxxxxxxxxxxxxxxxx

# ============================================
# Call configuration
# ============================================
# Fixed dial-out number
FIXED_TO_NUMBER=+16198597172

# Public URL (using ngrok)
PUBLIC_BASE_URL=https://abc123.ngrok.io

# ============================================
# Optional configuration
# ============================================
MEDIA_SERVICE_PORT=4001
USE_PYTHON_VAD=false
AI_AUDIO_GRPC_URL=localhost:50051
```

## FAQ

### Q1: What is the difference between API Key and Auth Token?

**Auth Token:**
- Account-level primary credential
- Used to call Twilio REST APIs (start calls, hang up, etc.)
- Set as `TWILIO_AUTH_TOKEN` in `.env`

**API Key:**
- Used to generate Access Tokens
- Access Tokens are used by clients (Web/Mobile)
- Set as `TWILIO_API_KEY` and `TWILIO_API_SECRET` in `.env`

### Q2: Why do I need a TwiML App?

A TwiML App defines behavior when the Web client connects:
- When the Web client calls `device.connect()`
- Twilio requests the URL configured in the TwiML App (`/twiml/webJoin`)
- Your service returns TwiML instructions on how to handle the connection (join Conference)

### Q3: What if I forgot the API Secret?

The API Secret is **only shown once at creation time** and cannot be viewed again.

**Solution:**
1. Delete the old API Key
2. Create a new API Key
3. Update `TWILIO_API_KEY` and `TWILIO_API_SECRET` in `.env`
4. Restart media-service

### Q4: Can I use Account SID and Auth Token to generate tokens?

**Not recommended**, because:
- Auth Token has overly broad permissions (can operate your entire account)
- If leaked to clients, it introduces security risks
- Best practice is to use API Key (finer-grained permissions)

### Q5: What needs to be updated when the ngrok URL changes?

Every time you restart ngrok, the URL changes. Update:

1. `PUBLIC_BASE_URL` in `.env`
2. Twilio Console → TwiML App → Voice URL
3. Restart media-service

## Security Recommendations

1. ✅ **Do not commit `.env` files to Git**
   - Already configured in `.gitignore`

2. ✅ **Rotate API Keys regularly**
   - Replace every 90 days

3. ✅ **Use environment variables**
   - Do not hardcode credentials in code

4. ✅ **Use HTTPS in production**
   - Twilio requires webhook URLs to use HTTPS

## Troubleshooting Steps

### 1. Check whether environment variables are loaded

Temporarily add this in `apps/media-service/src/index.js`:

```javascript
console.log('[DEBUG] Config loaded:', {
  hasApiKey: !!config.twilioApiKey,
  hasApiSecret: !!config.twilioApiSecret,
  hasTwimlAppSid: !!config.twilioTwimlAppSid
});
```

Expected output:
```
[DEBUG] Config loaded: { hasApiKey: true, hasApiSecret: true, hasTwimlAppSid: true }
```

### 2. Check token generation

Add logs in the `/token` endpoint:

```javascript
console.log('[DEBUG] Token params:', {
  accountSid: config.twilioAccountSid,
  apiKey: config.twilioApiKey,
  apiSecretLength: config.twilioApiSecret?.length,
  twimlAppSid: config.twilioTwimlAppSid
});
```

### 3. Verify TwiML App URL

Make sure the TwiML App Voice URL points to:
```
https://your-ngrok-url.ngrok.io/twiml/webJoin
```

You can test manually:
```bash
curl -X POST https://your-ngrok-url.ngrok.io/twiml/webJoin \
  -d "sessionId=test_session"
```

It should return TwiML XML.
